# Wkład do projektu

Jak wprowadzać zmiany i tworzyć PR-y:

1. Stwórz branch z opisową nazwą: `feature/<opis>` lub `fix/<opis>`.
2. Zrób małe, logiczne commity z opisowym komunikatem.
3. Otwórz Pull Request do `main` z opisem zmian i linkami do relewantnych issue.
4. Dodaj do PR krótkie kroki jak uruchomić zmiany lokalnie.

Standardy kodu
- Używaj TypeScript i typów dla publicznych API komponentów.
- Preferuj czytelność i proste propsy.
